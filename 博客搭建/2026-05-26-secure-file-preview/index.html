博客文件上传与密码预览功能记录

本文最后更新于 2026-05-26 14:18

背景

博客除了写文章，也可能需要放 PDF、Word 或学习资料。为了避免资料被直接下载，当前后台支持“安全上传”：

  • 原始文件加密保存
  • 生成带水印的预览图
  • 打开部分资料时需要输入密码

静态博客的边界

GitHub Pages 本质是静态网站，不能像完整后端系统那样做严格权限控制。前端可以隐藏下载入口、加水印、加密文件，但无法绝对阻止截图、录屏或浏览器缓存。

所以更合理的策略是：

  • 普通文章直接公开
  • 学习资料使用水印预览
  • 重要资料只放摘要，不上传源文件
  • 真正敏感文件不要放到公开仓库

使用方式

通过本地管理员后台上传资料：

然后访问：

发布时后台会触发构建、提交和推送，并监测 GitHub Pages 的部署进度。

总结

对个人博客来说，资料保护的目标是“降低误传和随意传播风险”，不是替代专业网盘或权限系统。把资料按公开程度分类，会比单纯依赖前端限制更可靠。
